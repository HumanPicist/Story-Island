# 日志 · 说明与模板

文件名：`day-第N天.md`，一个文件 = **一个周期**（默认一整天），内部按**时段**累加（`###` 小标题分段）。同一周期多次「推进」往里追加时段块；跨周期开新文件。
（**三层随世界·默认「天 / 小时 / HH:MM」**：页内三层＝`## 周期` → `### 时段` 小标题 → `- 〈刻度〉 事件` 行，各层单位由 [[世界设定]]「引擎参数·时间模型」定（史诗世界文件名即 `纪元-第N纪.md`、时段=年、刻度=关键日期）；frontmatter 周期序号字段一律 `周期`。详见宪法 §2.2「时间三层 / 单位总注」。下文一律拿「天 / 小时 / HH:MM」当默认范例写。）

注意：不要擅自改变格式。**跨天、每日第一次推进、或格式拿不准时，一律回看本说明的模板与下方〈格式铁律〉对照，不要参考最近几天的成稿**——成稿可能已漂移，照抄会越抄越偏。（新世界起初没有任何 day，第一天就照本模板和铁律写起。）

```markdown
---
类型: 日志
周期: 0
日期: ""
出场角色: []        # ["[[chr-甲]]"]
地点: []            # ["[[fact-某地]]"]
重大: false
骰点: []            # 如 ["08:40 相遇 D20=14 成功"]
更新于: ""
---

## 第N天 · YYYY-MM-DD

### 08:00–09:00
- 08:10 〔事件，有细节〕
  - **某角色**：「对话……」
  - **另一角色**（动作/语气）：「回话……」
- 08:40 〔并发的另一处〕

### 09:00–11:00
- ……

#### 今日延伸
- 展开：[[evt-某事件]]
- 触发大事件：[[big-某节点]]
- 动态事实更新：[[fact-某任务]]
```

> 每条带**刻度**（行首时刻标记）；一个时段内可多事件并发 + 对话。`###` 小标题＝**时段**（日常用钟点段如 `08:00–09:00`、或命名如 `### 下午`；史诗用 `### 早春` 之类）。
> 重点描述、细节、角色动作可以加粗表示，方便阅读重点。
> **对话格式**：`名字：「话」`，名字加粗，带动作/语气写 `名字（动作）：「话」`，连说两段 `名字：「话一」「话二」`。不用 `>` 引用块。

## 格式铁律（防漂移 · 每次写日志前过一遍，写完跑闸）

日志正文是**给人读的故事**，不是给引擎读的状态台账。下面五条是硬规矩，违反就是漂移。
**机器闸**：写完当天日志、动 `_state` 之前跑 `python3 08-脚本/lint_day.py`（默认查最新一天）——它把下面 #1–#3 的违规连同行号一并报出来（ERROR=红，必须改到绿；WARN=尾块建议）。这是 CLAUDE.md §4.1 写日志阶段 的硬步骤。（脚本住 `08-脚本/`；阈值/禁词可在 `08-脚本/调参面板.md` 零门槛改，见 `08-脚本/_说明.md`。）

1. **骰点只住 frontmatter。** 所有 `D20=…` 的完整列表写进 frontmatter 的 `骰点:`，**正文里不许出现骰值**——不许在小时块开头堆一坨 `（掷骰 D20(…)=…）`，也不许在句子里夹 `（D20=11）`。结果用人话写进叙述（"烧没往坏走，也没退"），不写"D20=11，持平"。
2. **正文不许出现系统/元词汇。** 引擎自己的裁决与追踪语只许活在 `_state` / frontmatter / evt·big 的分析段，**绝不进 day 正文**。禁例：`〔意义：…〕`、`〔清楚的新状态：…〕`、`〔设定级发现（grounded）〕`、`红线守旧界未缩`、`在退的平台/平台`、`Q6 悬置`、`钟仍在`、`measured` 之类。要表达"意义"，用角色当下的感受和具体动作去写，别贴标签。
3. **整点小结压成一句话快照。** 每个整点收尾**只写一句**当前现场（谁在哪、最紧的一件事），不逐角色列带骰值的状态条——那是 `_state` 在场快照的活儿，日志不替它记第二遍。
4. **范式是"动作·对白·感官细节"，不是任何一份旧日志。** 写法基线＝本铁律 #1–#3 本身——新世界第一天就照本模板与铁律写起。世界跑久后要找一份干净样例，看任一通过闸的日志即可（机器判定一律以 `lint_day.py` 绿为准）。**永远不以最近几天为参照**——最近几天可能正漂着。
5. **写完过闸。** `python3 08-脚本/lint_day.py` 红了就按它报的行号把正文改干净、重跑到绿，再写 `_state`。光靠"写前过一遍"压不住漂移——闸才压得住。

## 文风（承 [[世界设定]]·文风）

格式铁律（上）管"正文别漂成台账"；文风管"腔调别漂"。两者正交——格式有机器闸，文风没有，全靠回锚到规则本身。

写日志前对照 [[世界设定]] 的「文风 / 语言」段；**不参照最近几天的成稿**（防漂移，同上 §格式铁律 #4）。

本节留给「日志」这一层的笔法细化——随故事发展自填，暂留空。
