# 事件展开 · 说明与模板

文件名：`evt-标题.md`。只为**值得展开的事件**做完整记录（细节 + 对话）；一笔带过的不展开。

## 写法铁律（重要 · 别再漂）

evt 是**记录密度轴**——它只干一件事：把日志里一笔带过的那一场，**用过程、感官细节、完整对白沉浸式展开**，让人读着像在现场。它**不是**分析页、不是论证页。

- **只写「发生了什么」，不写「这意味着什么」。** 动作、对白、表情、气味、声音、温度、光线、角色当下的生理与心理反应——全要、要细。"这一场的意义 / 对某弧线的作用 / 撞了哪条线索 / 为什么重要 / 是不是反证点 / 从希望到冷账"——**全不要**，那些是后果与意义层，归配套的 [[big-*]]（**big 记后果与意义，evt 记过程质感**）。一个 evt 里出现「这一笔的意义」「弧线」「反证点」这类小节或句子，就是漂了，删。
- **小标题用场景 / 动作式，不用定义式。** ✓「水边那个人」「他蹲下去翻口袋」「回报」；✗「一、辨认的三层落空」「二、干预落地为半解」。小标题答"这一刻在演什么"，不答"这一段在论证什么"。
- **不写系统元语言**：正文里不出现"grounded / 降神 / 干预落地 / 掷骰 / D20 / 判定 / 概率 / 行为判定"等引擎词（骰值只进**日志** frontmatter 的 `骰点:`，evt 正文也不写）。结果一律用人话叙述（"压回去了""认不出"）。
- **配 big 时**：big 负责"后果量级与意义"，evt 只负责"过程质感"，两页双链互指。**意义内容只写进 big、不在 evt 复述**。
- **不配 big 时**：evt 仍只写过程；意义不必硬塞，留给日志「悬念」行或编年史去回望。

```markdown
---
类型: 事件展开
标题: ""
时间: ""            # "第N天 · YYYY-MM-DD · HH:MM"
发生于: ""          # "[[day-第N天]]"
出场角色: []        # ["[[chr-甲]]", "[[chr-乙]]"]
地点: []            # ["[[fact-某地]]"]
---

（完整经过、对白、细节——沉浸式展开那一场；**不写意义/分析层、不写系统元语言、小标题用场景动作式**。回链到对应日志小时块；配套 big 时双链。）
```
