# 07-面板 · 说明

这一层不是"世界内容"，是给**观察之神**看的一组只读可视化。

> 本页讲面板层**固定不变的逻辑**（故事发展面板 + 可派生面板的机制），**不含任何具体世界的面板内容**。模板默认不含派生面板；你的世界长出值得画的对象时，按下文〈派生面板（模式）〉新增、并把它补进本说明。

**命名约定**：「**面板**」是这一层**所有可视化的统称**，分两类：

- **故事发展面板（固定）** ＝ 总览。仓库骨架自带 `build_panel.py`，扫全库拼出一张总览（`故事岛-面板.html`，并内联进 `面板预览.md`），一眼看清此刻故事全貌与最近事件。**每个世界都有，第一次推进故事后即可创建。**
- **派生面板（可选 · 0～任意多）** ＝ 把某页 `fact-…` 的 `## 数据层` 渲成一张拟物 SVG（地图、物品栏、关系看板、状态条……随世界而定）。**新世界默认一张都没有**；当世界长出值得可视化的对象时，按下文「派生面板（模式）」新增。

## 文件

> **脚本与产物分家（重要约定）**：所有 `*.py`（`build_panel` 及任何 `build_<名>` 派生渲染器）统一住 **`08-脚本/`**（横切的"引擎脚本层"，连同三道格式闸 + 调参面板都在那；详见 `08-脚本/_说明.md`）。本文件夹 `07-面板/` 只留**产物**——`.html` / `.svg` / `面板预览.md`。下面提到的脚本一律去 `08-脚本/` 找。
> **注意**：Obsidian 不渲染 `.py`，在 Obsidian 里看 `08-脚本/` 是空白的属正常——要看/改脚本去本地文件夹（访达）用文本编辑器开。

- `build_panel.py`（在 `08-脚本/`）— 故事发展面板生成器。纯 python3、**零依赖**。扫描整座仓库的 .md，拼出一张自包含 HTML。**骨架自带，勿删。**
- `故事岛-面板.html`（在 `07-面板/`）— 产物。单文件、离线，双击即在浏览器打开。
- `面板预览.md`（在 `07-面板/`）— **单一汇总预览页**：页底是故事发展面板的内联 HTML（**阅读视图**显示）；有派生面板时，在其「派生面板嵌入区」依次嵌入各 `![[…svg]]`。由 `build_panel.py` 一并自动重建，**正文勿手改**（会被覆盖）。
  - 为什么不是 iframe：Obsidian 阅读视图用 DOMPurify 过滤——删 `<style>`、删 iframe 的 `app://`/`data:` src、删 `srcdoc`。所以本地 html 不能 iframe 内嵌，只能重渲成只用内联 `style=""` 的 HTML 写进笔记。
- `build_<名>.py`（在 `08-脚本/`）+ `<名>.svg`（在 `07-面板/`）— 派生面板渲染器与产物，**按需新增**（新世界起初没有）。

## 故事发展面板 · 刷新与查看

面板是**快照**，靠跑脚本更新：

```
python3 08-脚本/build_panel.py
```

**Live 机制**：已写进 [[CLAUDE|操作宪法]] §4.1「推进」收尾——每次推进写回 `_state` 后自动跑一次本脚本，面板始终跟世界同步（含后台无人值守推进）。想手动刷新就说一句"刷新面板"。

**查看入口**：

- **本地 `故事岛-面板.html`（高保真主力）**：任何环境的推进都重建它，浏览器里留个标签、刷新（Cmd-R）即见最新。
- **`面板预览.md`（Obsidian 内，单一入口）**：切「阅读视图」看故事发展面板（内联 HTML）；派生面板 SVG 在阅读/实时预览都直接显示。

**故事发展面板读了哪些**（世界无关、固定）：

| 区块 | 数据来源 |
| --- | --- |
| 世界时钟 + 氛围句 + 在场 | `_state.md`、最近 `day-` frontmatter |
| 在场角色卡（状态/位置/此刻/动机） | `_state` 在场快照 + `01-角色/chr-*.md` |
| 最近事件时间线 | 最近 `day-第N天.md` 的最后 3 个小时段 |
| 悬念与进行中的线 | `_state.md` 悬念区 |
| 动态事实 | `03-动态事实/fact-*.md` |
| 大事件 | `05-大事件/big-*.md` |
| 编年史 | `06-编年史/编年史.md` 最新一卷 |
| 最近骰点 | 最近 `day-` frontmatter 的 `骰点` |
| 待裁决 / 未结干预（高亮告警） | `_state.md` 对应区，非空才显示 |

## 派生面板（模式 · 加新视图照此办）

把某页 `fact-…` 的 `## 数据层`（契约见 `03-动态事实/_说明.md`）渲成一张随推进自动更新的拟物 SVG。

1. **渲染器** = 一个零依赖 `build_<名>.py`：读该 fact 页的 `## 数据层` YAML（用一个够用的 YAML 子集解析器）→ 拼 SVG 字符串 → 写 `<名>.svg`（预览由 `build_panel.py` 汇总嵌入 `面板预览.md`，渲染器自己不写预览页）。
2. **挂进自动重建**：`build_panel.py` 末尾有个循环，逐个 `import` 这些模块并跑 `main()`，**各自 try、报错不阻断**面板/推进。新增渲染器只需把模块名加进那个元组，并在 `面板预览.md` 模板的「派生面板嵌入区」加一行 `![[<名>.svg]]`。
3. 于是"每次推进跑 `build_panel`"（§4.1 重建阶段）这条现成钩子，把所有派生面板一并重绘——**零额外 token、后台无人值守也更新**。

**新增一张派生面板 = 改三处、务必同步**（默认 `build_panel.py` 出厂时**派生元组 `for _mod in ()` 为空、`PREVIEW_NOTE` 的「派生面板嵌入区」也空**，正等着填）：

1. **建** `build_<名>.py`：读对应 `fact-…` 的 `## 数据层` → 渲 SVG → 写 `<名>.svg`（遵守下面「增长安全」铁规）。
2. **登记到自动重建**：在 `build_panel.py` 的派生元组 `for _mod in ( … )` 里加上 `"build_<名>"`。
3. **登记到预览**：在 `build_panel.py` 的 `PREVIEW_NOTE`「派生面板嵌入区」加 `## <emoji> <名>面板` + 空行 + `![[<名>.svg]]`。

> 漏掉第 2 步 → 这张面板不随推进自动重建；漏掉第 3 步 → 预览页里看不到它。**注意分工**：日常"改 fact 数据层"只改 fact 页、**不动脚本**；只有"**新增/删除一整张面板**"才需要动 `build_panel.py` 这两处。

**为什么要数据层这层中介**：正文是散文、脚本读不准；数据层把"已写明"的量/坐标/状态结构化出来，脚本才能确定性重绘。所以**自动更新的前提是数据层与正文同步**（推进时一并改，§4.1 更新对象阶段）；坐标这类正文给不出绝对值的，由人在推进时填个示意值。

**★ 增长安全（格式维护铁规，加新面板务必遵守）**：渲染器**绝不写死容量**——布局由**数据条数推导**、**画布按需增高**、文本用「**按容器像素宽折行 + 限行省略**」。卡片/便签多了就自动加行加高、或走**固定列宽的网格**往下排，**绝不让单元变窄而文字溢出**。任何"随推进条目变多就错位/溢出/越界/`IndexError`"都属违反本规，须改成数据驱动。

**维护旋钮**：增删内容＝改对应 fact 页的数据层即重绘；视觉细节（配色/图标/分档/连线…）在各 `build_*.py` 顶部的常量与小函数里改。**具体世界内容只活在数据层、随世界生灭，不写进说明、更不写进宪法——这里只记不变的"怎么做"。**

## 约定

- 面板**只读**：读仓库、不写世界（除了输出自己的 html / svg / 预览页）。
- 解析全在本机磁盘完成，**不耗推进的上下文 / token**。
- 脚本按相对路径定位仓库根（`__file__` 的上两级），换机器 / 换路径都不用改。
- 这一层不参与世界因果，推进时不必把它当"现实"读。
