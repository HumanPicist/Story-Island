# `_God` · 说明

**这是「观察之神」的后台工作间**——放元层东西的地方：可移植的种子提示词、范例世界、以及任何你不想混进世界正文的幕后材料。**它不是世界内容**，引擎在推进/日结时**一律不读、不写、不索引**这里（宪法 附录 B「私有区」：根目录下以下划线开头的**文件夹**默认隔离）。所以这里随便放、随便乱，不会污染世界、也不耗推进的上下文。

> 命名上 `_God` 以下划线开头 = 私有文件夹标记。世界跑起来时，这一层对引擎是透明的；只有你（或被你明确点进来的 LLM）才会翻它。日后你也可自建其他下划线开头的文件夹作为私有区。

## 里面有什么

| 条目 | 是什么 |
|---|---|
| `环境与首次设置.md` | **首次上手 / 环境指南**：讲清三层依赖（Obsidian / 引擎 CLI / Python）与"自举死结"，给一段「首次自检提示词」让引擎替用户查装 Python，外加三平台手动装 Python 速查、装引擎 CLI 指引、Claudian 的 PATH/别名排错、无 Python 降级真相。给人读、运行非必需；根目录 README 挂了入口。 |
| `后台定时运行指南.md` | **无人值守操作指南**：教用户用 Codex / Claude cowork 的定时任务，让世界自动推进 / 体检 / 重绘编年史。含可直接复制的「任务提示词」（A 推进 / B 体检 / C 重绘）+ 建任务步骤。给人读、运行非必需；根目录 README 挂了入口。后台运行时它**也会被引擎写进来**（体检报告落 `_God/体检报告.md`）。 |
| `LICENSE` | 本仓库许可正本（CC BY-NC-SA 4.0）。**特意放 `_God` 而非根目录**——免得在 Obsidian 主视图碍眼；README 与 `_index` 链向它。（代价：GitHub 不会自动识别根目录许可徽章，README 已写明许可。） |
| `第三方组件与许可.md` | 随附第三方插件/主题（Claudian·MIT / Front Matter Title·GPL-3.0 / Retroma·MIT）的来源与许可清单；各组件 LICENSE 原文随其文件存于 `.obsidian/`。 |
| `纲领提示词.md` | **哲学种子**：把这套系统的精神（反收敛、三角色、时间一等公民、世界要生长…）压成一份可移植提示词。**可选参考、非运行必需**——这个仓库本身 clone 即用，根目录 `CLAUDE.md` 就是长成的树。（注：该种子早于当前这套机器层，细节一律以根 `CLAUDE.md` 为准。） |
| `示例世界/` | **极简范例**：一个填好的迷你世界切片，给新用户看一眼"填出来的角色 / 日志 / 事实长什么样"。**纯内容、不含任何脚本 / 宪法副本**（杜绝"改错了真宪法"的混淆），与活仓库的空模板互不干扰。（首次拿到模板时可能尚未放入。） |

## 约定

- **改这里不影响运行中的世界**：种子 / 范例 / 备忘随便整理；引擎不看。
- **唯一会被引擎引用的系统文件是根目录的 `CLAUDE.md` / `_state.md` / `_index.md` / `_log.md`**，都不在这。这里纯属"幕后"。
- **想把这套系统给别人**：直接给整个仓库（clone 即用、`当前周期:0` 静候创世）。`纲领提示词.md` 仅在你想"从精神重新长一套"时才用得上。
- **别在 `_God/` 里放第二份脚本或宪法**：那会变成和活仓库并存的漂移源（曾经的教训）。机器只有一套，住根目录的 `08-脚本/` 与 `CLAUDE.md`。
