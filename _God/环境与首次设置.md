# 环境与首次设置

> **一句话**：这套系统要三样东西——① Obsidian（看 / 编辑）、② 一个 AI 引擎 CLI（Claude Code / Codex——跑"世界引擎"的那层躯壳；真正的引擎＝模型 + `CLAUDE.md` 宪法）、③ Python 3（跑三道格式闸 + 面板）。前两样得你手动装好（系统没法替你装"跟它对话的入口"），**装好引擎后，Python 这步可以让引擎替你自动搞定**。

---

## 先认清一个"自举死结"（最重要）

这套系统的"安装器"，本质上就是你的 **AI 引擎**——它能跑命令、能帮你装东西。但**引擎自己恰恰是唯一不能自动装的**：你得先有它，它才能替你装别的。就像"你需要一把钥匙才能进门拿钥匙"。

所以诚实地分两层：

| 依赖                              | 谁需要                      | 能自动装吗  | 怎么办                                   |
| ------------------------------- | ------------------------ | ------ | ------------------------------------- |
| **Obsidian**                    | 所有人                      | ❌      | 普通桌面 app，去 obsidian.md 下载双击装（非技术用户也会） |
| **引擎 CLI**（Claude Code 或 Codex） | 所有人                      | ❌（自举）  | 手动装一次，见下方〈装LLM CLI〉或者问AI让它引导你安装       |
| **Python 3**                    | 所有人                      | ✅      | **装好引擎后，让引擎替你查 + 装**（见下方〈首次自检提示词〉）    |
| Claudian 插件                     | 只在 Obsidian 内用引擎的人       | 已随仓库附带 | 它**只是前端**，仍要先把引擎 CLI 装好、且在 PATH 上     |
| Node.js                         | 仅当你用 npm 方式装 Claude Code | 视情况    | 引擎 CLI 的"原生安装器"通常不需要 Node；用 npm 装才需要  |

**结论**：真正要你亲手做的只有「装 Obsidian + 装引擎 CLI」两件——而这本来就是用这套系统的入场券，不算额外负担。**Python 及之后的一切，交给引擎。**

---

## 首次自检提示词（最省事的路）

装好引擎 CLI 后，第一次用，**把下面这段整段粘给你的引擎**（Claude Code / Codex / Claudian 都行），让它替你把环境过一遍：

```text
我刚拿到「故事岛」这个仓库，想先确认运行环境齐不齐。请以这个仓库文件夹为工作目录，帮我做一次首次环境自检（这一步只装环境，先别创世、别推进）：

1. 先报一下我的系统：macOS / Windows / Linux。
2. 检查 Python 3：跑 python3 --version（Windows 上也试 python --version 和 py --version）。
3. 若没装 Python 3，按我的系统用最稳妥的方式帮我装——动手前先告诉我你要跑什么命令、要不要管理员权限，等我点头：
   - macOS：优先 xcode-select --install（会弹出图形安装窗，我点「安装」即可）；我若有 Homebrew，brew install python 也行。
   - Windows：winget install Python.Python.3；或让我去 python.org 下安装包，并提醒我勾选「Add Python to PATH」。
   - Linux：用我发行版的包管理器，例如 sudo apt install python3。
4. 装好后跑 python3 08-脚本/lint_state.py 验证：只要它能正常执行（哪怕提示世界未创世之类的内容问题），就说明 Python 环境通了。
5. 最后用大白话告诉我：环境齐了没、还差什么、下一步干嘛。
```

引擎会按你的系统自适应处理、装完自己验证、说人话汇报。**通过后，再开始「创世」。**

## 手动装 Python 3（不想走提示词，或引擎装不动时）

> 本仓库脚本是**纯标准库、零第三方依赖**——只要有 Python 3，不用 `pip` 装任何包。

**macOS**（任选其一，从上到下越来越"开发者向"）：
- `xcode-select --install` —— 终端跑一句，弹图形窗点「安装」；装的是 Xcode 命令行工具，自带 `python3`。**最省心。**
- 去 [python.org](https://www.python.org/downloads/) 下 macOS 安装包，双击装。
- 有 [Homebrew](https://brew.sh) 的话：`brew install python`。

**Windows**（任选其一）：
- `winget install Python.Python.3` —— 在「终端 / PowerShell」里一句搞定。
- 去 [python.org](https://www.python.org/downloads/) 下安装包，**安装时务必勾选「Add Python to PATH」**（否则命令行找不到它）。
- 或微软商店搜 "Python 3" 安装。
- 注意：Windows 上命令多是 `python` 或 `py`（不是 `python3`）。

**Linux**：多数发行版自带；没有就 `sudo apt install python3`（Debian/Ubuntu）/ `sudo dnf install python3`（Fedora）等。

装完验证：在仓库文件夹里跑 `python3 08-脚本/lint_state.py`，能执行即通。

## 装LLM CLI（引擎的"躯壳"·让模型能动手）

> 严格说，"世界引擎"的**智能在 LLM 模型里、规则在 `CLAUDE.md` 操作宪法里**；CLI 是把这两者**跑起来、并给它一双手**（读写你的仓库、跑脚本、连续多步行动）的躯壳。所以它不是"本体"，但**没有它，模型碰不到你的世界**。Claudian 则只是这层躯壳的一个 Obsidian 窗口。


两个主流选择，装哪个都行（也可都装）：

- **Claude Code**：按 [Anthropic 官方文档](https://docs.claude.com/en/docs/claude-code) 装（通常是一条官方安装命令，或 `npm install -g @anthropic-ai/claude-code` —— 后者需要 Node.js 18+）。
- **Codex**：按 OpenAI 官方指引装其 app 或 CLI。

> 具体命令以**官方文档最新版**为准（版本会变，我不在这写死可能过期的命令）。装完在终端跑一下它的 `--version` 确认在 PATH 上。


**_Tips: 如果实在不想装CLI,并且接受不在obsidian内进行全部交互，可以简化使用：Codex或Claude客户端的cowork/code模式，开一个project/对话，并把仓库文件夹挂载为工作区，也可完成故事世界运转的交互。_**

**_Tips: 想在Claudian中接入GLM、Deepseek等其他开放模型的用户可自行查询配置方法_**

## 用 Claudian 的人，额外注意

Claudian（`realclaudian`）是 Obsidian 里的一个**前端面板**——它把 `claude` / `codex` 这些 CLI 嵌进右侧栏，**自己不含引擎**，靠 `spawn` 去调你系统里装好的 CLI。所以：

- **前提**：上面那个引擎 CLI 必须已装好、且在 PATH 上。Claudian 找不到它就用不了。
- **PATH 坑（最常见报错）**：你在「终端」里能跑 `claude`，但 Claudian 报「找不到 claude / command not found」——这是因为图形 app（Obsidian）启动时的 PATH 往往比你终端里的窄，看不到 `~/.local/bin`、`/opt/homebrew/bin` 这些。解法：① 在 Claudian 设置里直接**填 CLI 的完整路径**（如 `/Users/你/.local/bin/claude`，你的路径用 `which claude` 查）；② 或把 CLI 装到系统标准位置；③ 或从终端启动 Obsidian 让它继承你的 PATH。
- **别名（alias）不算数**：如果你的 `codex` 只是 shell 里的一个 `alias`（不是 PATH 上的真可执行文件），Claudian 和定时任务用 `spawn` 调用时**看不到别名**。要么装成真正在 PATH 上的可执行文件，要么在设置里给它的全路径。

## 没 Python，世界还能跑吗？（降级真相，不推荐）

能，但残废。Python 只在两处发力：**三道格式闸**（写完日志 / 记忆 / `_state` 后的自动校验）和**面板生成器**（`07-面板/` 那张 HTML 仪表盘）。没有 Python：

- 引擎仍能用自然语言推进世界、写文件——世界不会停。
- 但**失去自动格式校验**（容易悄悄积累格式漂移）和**自动面板**（没有那张可视化总览）。

所以：临时应急可以无 Python 硬跑，**但正经用，还是把 Python 装上**——它零依赖、装起来最省事的那条路（`xcode-select --install` / `winget`）几分钟就好。

## 一分钟自检清单

开跑前对一遍：

- [ ] Obsidian 能打开这个仓库
- [ ] 终端里 `python3 --version`（或 `python` / `py`）有输出
- [ ] 在仓库里 `python3 08-脚本/lint_state.py` 能执行
- [ ] 引擎 CLI（`claude` / `codex`）`--version` 有输出、在 PATH 上
- [ ]（用 Claudian 的话）右侧栏里它能正常调起引擎、不报找不到 CLI

全绿 → 跟引擎说「**创世**」，正式开始。卡在哪一条，就把那条对应的上面那节读一下，或直接把报错贴给引擎问。
