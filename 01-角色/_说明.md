# 角色 · 说明与模板

文件名：`chr-名字.md`。**创世时不用手搓**——本文件夹已铺好两张可填模板页 [[chr-角色一]] / [[chr-角色二]]，像填 [[世界设定]] 一样开页即填（改名、填四段、复制可加更多、用不到就删）。下面这份是它们的 **schema 依据**（字段 + 段落约定），写法拿不准时回来对照。

> **「角色」＝施动者，不一定是人。** 凡 `推进` 时引擎要"逐个独立判定它会做什么"的，都用这套 chr- schema 建页——可以是**个体**（人 / 宠物 / AI），也可以是**集体**（阵营 / 组织 / 势力 / 一个文明）。集体同样有"核心动机"（其驱动意志）与"记忆"（其近期与塑造性历史）。`类型` 字段一律写 `角色`（个体或集体都用同一套 schema；也可写 `阵营` 等，脚本只按"施动者"统一逐个判定、不在意这个标签；范例见 `_God/示例世界/史诗-星海纪/chr-结晶教团.md`）。**分界线是能动性**：会**行动**的 → 角色（`chr-`）；被动被追踪的（地点 / 资源 / 谜团 / 物件）→ 动态事实（`fact-`，见 `03-动态事实/_说明.md`）。两者并存。

```markdown
---
类型: 角色
名称: ""
别名: []
状态: 活跃          # 活跃 / 离场 / 失踪 / 死亡
位置: ""            # "[[fact-某地]]"
关系: []            # ["[[chr-某人]]"]
首次登场: ""        # "[[day-第1天]]"
更新于: ""          # 世界内时间戳
---

## 角色设定
（外貌、性格、背景、当前处境、**核心动机**——推进时角色靠它自行决策。）

## 短期记忆
（最近还在影响 ta 的事，带 `[第N天 · 日期 · HH:MM]`，链接到日志/事件。会滚动维护。）

## 重要记忆
（塑造了 ta、长期起作用的关键事件**及其影响**。）

## 补充
（口癖、秘密、伏笔、不宜字段化的质感。）
```

> 动机写得越具体，高自主模拟里这个角色越"活"。frontmatter 里所有 wikilink 记得加引号。

## 重要记忆 · 覆写式纪律与字数闸（canonical）

重要记忆是**沉淀下来的耐久核**，不是第二本日志。它最危险的发胖方式是「按天 append」——同一条弧每天往尾巴上摞一条带日期的 beat，第 5 天看着还行，第 30 天就是每条线三十条。这套纪律把它焊死。机器闸＝ `08-脚本/lint_chr.py`（日结记忆结算后跑 `python3 08-脚本/lint_chr.py`，红了改到绿，见 CLAUDE.md §4.7·3）；脚本住 `08-脚本/`，下列字数闸/禁词可在 `08-脚本/调参面板.md` 零门槛改（见 `08-脚本/_说明.md`）。

**固定三子段（每个角色都建，结构对齐才可被闸解析）**：

```markdown
## 重要记忆
（一句话前史：塑造 ta 的登场前关键往事，写一次、基本不动。）

### 关键事件·节点
- [第N天 · 日期] 史实锚点……

### 关系变化
- [第N天] 与某人：当前关系态……

### 心境·态度演变（角色弧线）
- [第N天] 某条弧：当前态度态……
```

**两种写法（关键！）**：

| 子段 | 性质 | 写法 | 字数闸（起步值，松紧在 `08-脚本/调参面板.md` 改） |
|---|---|---|---|
| **关键事件·节点** | 带日期的**史实锚点** | append-but-tight：一天/一大事一条，只记"durably 发生了什么+为何塑造"，过程细节归 日志/evt/big（回链）；**只压字、不删**（§5·3） | ≤约 **200 字/条** |
| **关系变化** | **当前关系态** | **覆写式**：一对关系**一段**，关系有移动就**改写那段**、不 append 新 beat | ≤约 **6 段**、≤约 **140 字/段** |
| **心境·态度演变（弧线）** | **当前态度态** | **覆写式**：一条弧**一段常驻线** + 最多两三个带日期的 pivot 短句；态度移动改写当前线、不每天摞 | ≤约 **8 条**、≤约 **160 字/条** |

> **覆写式**＝和 `_state` 当前态快照、动态事实「当前状态」同一个纪律：写的是**此刻这条关系/这条弧停在哪**，不是它一路怎么走来的流水（流水在日志里）。
>
> **字数不是行数**：md 里一行可 20 字可 200 字，按行卡等于没卡——闸一律数**字数**（剥掉 `←` 指针尾、`**`、`[时间戳]` 后的净字数）。
>
> **反收敛护栏**：闸的语义是 consolidate（合并/改写/压缩），**不是 amputate**。史实一条不丢、长期默认不删；cap 设得宽是故意的——别为了达标把角色的摩擦、矛盾、未完成磨成讨好的整洁条目（那是本系统最反对的收敛）。卡的只是"按天摞"和"跨段同义复述"两种机械冗余。
>
> **反元词（人话不是台账）**：记忆正文禁引擎记账词（`Q\d`、`悬置`、`measured/declared/grounded`、`红线守界`、`未兑现`、`钟仍在`、`〔意义…`等）——记忆是给人读的，元词用角色的感受/动作去写，同日志 §格式铁律 #2。但**为记忆层裁窄**：放过 `指针`/`活跃面`/`降档`（短期记忆 schema 就叫「要点+指针」，它们在这里合法）。`lint_chr.py` 以 ERROR 查、与字数检查并列，默认词表 `META_WORDS` 在 `08-脚本/lint_chr.py`，可在 `08-脚本/调参面板.md` 的「记忆·额外禁词 / 记忆·豁免词」零门槛增删（基线不被清空）。这是世界设定「文风」那条"避免系统/元词汇污染"在角色记忆层的牙齿。

**短期记忆**：滚动窗口（当天+前两天）+ 按字数压，细则在 CLAUDE.md §4.7·C。同由 `lint_chr.py` 查，但**两种严重度**：**每条 ≤约 150 字 = ERROR**（闸只在日结跑，那时该已压成「要点+指针」，报红＝这次日结没压到位）；**条数 ≈24–30 = WARN**（§4.7 软性 judgment 优先，日结前尖峰正常、不阻断）。字数是 ERROR 因为它是确定性的、压没压得动一眼可判；条数留软，避免为达标硬删该留的未了情。
