# 动态事实 · 说明与模板

**每次故事推进后，涉及到动态事实的更新，一定要仔细核查动态事实页正文与数据层是否和最新日志和世界状态同步。如有脱节必须更新。**

文件名：`fact-名称.md`。需要长期跟踪进度的东西：场景、道具、物品/消耗、任务、图鉴、地图等。

## 写法铁律（反机械化 · 意义优先 · 上限阈值 —— 重要 · 别再漂）

动态事实页是**人对世界的当前理解**，不是第二份日志。每次推进更新它，**不是机械追加一条流水**，而是：

1. **意义层诠释，不是事件搬运**：写"这件事把我们的理解变成了什么"，不写"几点几分发生了什么"（后者归日志 `day-*`）。**正文与数据层都不出现骰值（`D20=…`、`行为判定=…`）、不照搬日志原文逐时叙述**——骰值只在日志 frontmatter。
2. **结合 · 整合 · 修正既有**：新信息要回去和已有判断**合并、整合、修正**——推翻旧推论就改掉它、印证就加固它、同母题就并成一条，而**不是在末尾摞一条新的**。「当前状态」是覆写式的"此刻认知快照"。
3. **上限阈值（软性 · judgment 优先 · 超限即压缩整合）**：
   - 「进度时间线·活跃」＝**关键节点**，**≤ 6–8 条**人话里程碑（带 `day/evt/big` 链接，逐时细节回日志查），**不逐事件**；
   - 数据层「进展」**≤ 12 条**里程碑（去骰值、同母题合并）；
   - 每条「线索」注 **≤ 3 行（约 80 字）**；数据层「总判」**≤ 约 150 字**。
4. **超限即压缩整合**（日结 §4.7 / 盘点 §4.4 滚动触发）：合并同母题、把旧细节**概括成一句总结**、留重要节点、过程条降档进归档面（只移不删）。**目标是页面随世界变老而"理解更清晰"，不是"记录更冗长"。**
5. **按对象性质适配（元逻辑不机械套用）**：上述上限针对的是**机械增殖**（逐事件流水、堆叠叙述、骰点搬运），不是一刀切砍内容。各页的"意义层"随子类不同——**任务/侦查页**＝推论与疑惑；**物品/消耗页**＝当前家底的准确快照（**物品件数由世界实际决定、按实照列不强行砍**，只把每条注压简洁、把逐次取水/消耗/补料等流水收进关键节点）；**地图页**＝当前地理认知（**地理点位按实照列**，与侦查页**去重**——异常史/推理/辨认经过只留指针指向对应任务页、不在地图页复述）。判断准则：**砍的是冗余与机械，不是世界的真实清单。**

## 两层结构（防膨胀 · 详见宪法附录 F）

动态事实是唯一"既必读、又只增"的对象，故拆成两个文件、两层读取：

- **`fact-名称.md`＝活跃面（必读）**：当前状态 ＋ 已知线索/疑问（任务类）＋ `## 进度时间线·活跃`（近3天窗口）＋ `## 数据层`。页尾留一行指针 → `[[fact-名称-归档]]`。
- **`09-归档/fact-名称-归档.md`＝归档面（默认不读·指针触发回查·只移不删）**：放在根目录 `09-归档/`（系统冷存储区，**不在** `03-动态事实/`，否则会被面板当活跃 fact 渲染）。内含 `## 进度时间线·归档`，收降档下来的旧条目，**时间正序**（最旧在上、与活跃面续得上）、合并同母题。frontmatter `类型: 动态事实归档`。

**降档规则**（先升级、后降档，由 §4.7 日结 / §4.4 盘点 滚动执行）：进度时间线条目早于「当周期+前两周期」3周期窗口（`当前周期 − 条目周期 ≥ 3`，默认即"当天+前两天"3天）**且** 信息已被活跃面（当前状态/线索/疑问/数据层）吸收 → 剪到归档面；**线索汇总、疑问、当前状态永不降档**（它们是活跃推理面，不是流水）。归档面只增不删（历史兜底，宪法 §5·3）。子类只有寥寥几条时间线的小 fact，可暂不建归档文件，超窗口再建。

```markdown
---
类型: 动态事实
子类: ""            # 场景 / 道具 / 物品 / 任务 / 图鉴 / 地图 / 其它
名称: ""
状态: ""            # 进行中 / 已完成 / 已损毁 / 库存:3 …
更新于: ""
---

## 当前状态
（此刻是什么样。覆写式·权威·精炼。）

## 已知线索汇总 / 疑问   ← 任务/侦查类才有·活跃推理面·永不降档

## 进度时间线·活跃        ← 近3天窗口 + 仍开放的直接证据锚
- [第N天 · YYYY-MM-DD · HH:MM] 〔变化〕 ← 来源：[[day-第N天]] / [[evt-…]]
（每次变化追加一条；出窗口且已被活跃面吸收的旧条，由日结/盘点降档到归档面。）

## 数据层               ← 见下（机读当前态镜像）

> 更早进度时间线见 [[fact-名称-归档]]（默认不读·指针触发回查）
```

归档文件模板（存于 `09-归档/fact-名称-归档.md`）：

```markdown
---
类型: 动态事实归档
名称: ""            # 同主页名称
---
# 名称 · 进度时间线归档（默认不读）

> 宪法附录 F：活跃面满 3 天窗口时由日结/盘点把出窗口且已被吸收的旧条剪到此处。
> 时间正序、只增不删；不进任何推进读取层，仅指针牵到久远史实时翻。

## 进度时间线·归档
- [第1天 · …] 〔旧条〕 ← 来源：…
```

## 数据层（机读镜像）

需要做**可视化**（拟物 SVG / 原生看板）或被脚本读取的页，在正文末尾加一个 `## 数据层`，用一个 ```yaml``` 块把「当前状态」里**已写明**的东西结构化出来。它是给机器读的**正文派生镜像**——不是新事实、不是历史。

铁规：

- **正文主权**：只镜像正文已说的；冲突信正文，**绝不在数据层编造正文没有的数字/坐标**。
- **随推进同步**：推进改了这页正文（量/状态/进度），**同一步一并改数据层**（宪法 §4.1·9）——否则视图会悄悄变旧、和正文漂移。
- **非历史**：历史永远以「进度时间线·活跃」+「归档面」兜底；数据层只反映"此刻"，可被覆盖。**「进展」一类的里程碑数组按线索开/闭控量**（同母题合并、所属线索了结才精简），**不随进度时间线的3天窗口降档**——它驱动线索板可视化、是开放侦查面镜像，与「线索/疑问」同进退；完整逐事件史去归档面查。
- **稀疏可空 + 引号**：填不出就留空；字符串含 `:`、`[`、`→` 或逗号时加引号（零依赖解析器与 PyYAML 都能读）。

字段名随对象与你将来加的渲染器自定，由该渲染器约定（如某物品页可能用 `物品`/`缺失`、某地图页可能用 `点位`/`迷雾`、某线索页可能用 `线索`/`疑问`/`进展`…）；渲染与"怎么加一张派生视图"见 `07-面板/_说明.md`（模板默认不含任何派生面板，世界长出值得画的对象时再加）。例（任意"物品/消耗"页节选）：

```yaml
版本: 1
物品:
  - { 名: "示例物资甲", 量: "约3个", 满量: 5, 类: 示例类, 状态: 可用, 注: "一句话注释" }
缺失: ["尚缺之物A", "尚缺之物B"]
```
