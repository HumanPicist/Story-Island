# 故事岛 · 自主世界模拟引擎（模板）

> 你用自然语言搭一个世界（设定 + 角色），一个 LLM 引擎**按周期推进**它（写实世界按小时/天、史诗世界按年/纪元——你在创世时定）——逐角色独立判定、关键处掷骰、忠实记录，可无人值守长跑。**它模拟一个会自行其是的世界，不写讨好你的故事。**
>
> 这个仓库文件夹是一个**空模板**：clone 下来填上你的世界即可开跑。当前 `当前周期: 0`（未创世）。

## 需要什么（基础设施）

- **Obsidian** — 当作"世界的现实"来读写这个仓库（写进文件的＝发生过的事）。
- **Python 3** — 跑三道格式闸 + 面板生成器（**零依赖**，纯标准库，不用 pip 装任何东西）。**没装也别慌**：装好LLM引擎后让它替你检查并安装，见 [[环境与首次设置]]。
- **一个会把 `CLAUDE.md` 当系统指令的 LLM 引擎** ＝ 你的"世界引擎"。任何能①读 `CLAUDE.md` 当操作宪法、②读写本仓库文件、③跑 python 的环境都行：Claude Code、Codex、Obsidian 里的 Claudian 插件、Claude客户端的Cowork模式 等。（注：**Claudian 只是前端**，仍需先把 Claude Code / Codex 的 CLI 装好、且在 PATH 上——细节见 [[环境与首次设置]]。）
- **（可选）定时任务** — 让引擎每小时（或每天）自动「推进」，世界无人值守地往前跑；还能定时体检、定时重绘编年史。**怎么挂、附可直接复制的提示词 → 见 [[后台定时运行指南]]**（`_God/后台定时运行指南.md`）。

> **第一次用 / 环境装不上？** → **[[环境与首次设置]]**（`_God/环境与首次设置.md`）：一段「自检提示词」让引擎替你把环境过一遍，外加三平台手动装 Python 速查、装引擎 CLI 指引、Claudian 排错。

## 三分钟创世

1. 跟引擎说一句「**创世**」（或「开张」）——它会**逐项问你**：世界名、设定、历法、**时间尺度**（按小时/天，还是按年/纪元）、基调、红线、起始角色、开场局面；你答，它落进文件（**它只问与转录，绝不替你编**——创世归你）。
   - 想自己来：直接填 [[世界设定]] + 至少一张角色模板 [[chr-角色一]]（**尤其核心动机**那句），填好对引擎说「开张」。
2. 引擎把世界时钟拨到第 1 天，世界开张。
3. 之后说「**推进**」（或「过一天」「推进到傍晚」），世界就往前走。

## 时间走多快，你说了算

世界默认**按天、按小时**走——像写实电影。但一次「推进」也可以跨**一年、一个世纪**：

- **写实**（一推进 ≈ 1 小时）：荒岛求生、公寓里的一夜、一场谈判。
- **编年**（一推进 ≈ 1 天 / 1 年）：一个家族三代、一座城的兴衰。
- **史诗**（一推进 ≈ 1 世纪 / 1 纪元）：一个文明、一片星海的演化。

创世时引擎会问你"一**周期**＝多大"——从此日志、面板、时间戳都按你的尺度走（`第3天 · 14:00` 或 `第3纪 · 银河历4112 · 晚春` 都行）。**写实世界什么都不用改**，默认就是按天、按小时。

## 七个动词（怎么驱动世界）

| 你说 | 引擎做 |
|---|---|
| 推进 / 继续 / 过X小时 / 过一天 | **推进**——世界往前走：逐角色判定 + 掷骰，写进日志 |
| 我想让… / 干预： | **干预**——把你的意图落成世界内合理因果（不降神） |
| 把那件事详细写写 | **展开**——某一场扩成完整记录 |
| 盘一下 / 现在什么状况 | **盘点**——体检：状态 / 线索 / 记忆 / 膨胀 |
| 写段编年史 | **编年**——跨期文学性回望 |
| 日结 / 收一下今天 | **日结**——每日故事 + 记忆结算（跨天自动跑） |
| X怎样了 / 这条线到哪了 | **检索**——读库作答、给出处 |

## 配套环境（插件 · 主题 · 显示名）

仓库随附一套配好的 Obsidian 环境，clone 即用：

- **Claudian 插件**（`realclaudian`）— 把 Claude Code / Codex 等编程 agent **嵌进 Obsidian 右侧栏**，直接以本仓库为工作目录读写文件、跑脚本。**这就是你的"世界引擎"入口**——打开仓库默认右侧就是它，对它说「创世」「推进」即可。
- **Front Matter Title 插件** — 让文件在 Obsidian 里**显示 frontmatter 的 `显示名` 而非文件名**。所以 `CLAUDE.md` 显示成"引擎规则"、本页显示成"上手指南"、`_state.md` 显示成"当前状态"：文件名保持机械稳定（脚本 / 双链不断），人眼看到的是好读的名字。**你的日志 / 角色页也可加一行 `显示名:` 让标题更直观。** 和LLM对话时，建议使用页面原名（文件名）
- **主题 Retroma · 浅色** — 外观已固定为**浅色**（`appearance.json` 的 `theme: moonstone`），且**随仓库走**：每个 vault 自带外观、不跟你其它 vault 的设置变，这套主题推荐浅色，主题色可自定义。**想换随你**：Obsidian 设置 → 外观里改主题 / 深浅 / 关掉 `精简骰点` snippet 都行。
